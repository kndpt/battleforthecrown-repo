# refactor-shared — run 2026-06-18b

## Thème : logic/ — suppression casts as-number superflus + import dupliqué

### Audit

Package shared en bon état général. Toutes les fonctions pures ont des consumer tests backend.
Seul problème identifié : `logic/building-cost.ts` et `logic/travel-time.ts`.

- `building-cost.ts` : deux imports séparés depuis `'../village/buildings'` + 2 casts `as number` sur `getStrategyBonusValue(...)` (clés `buildingCostReduction`, `constructionSpeedBonus`).
- `travel-time.ts` : 1 cast `as number` sur `getStrategyBonusValue(strategy, "armySpeedBonus")`.

Casts superflus car `getStrategyBonusValue<K>` retourne `Required<StrategyBonus>[K]` — TypeScript résout directement en `number` pour ces clés literales.

Candidats écartés :
- `auth/schemas.ts` : déjà testé dans backend `auth-schemas.spec.ts`.
- test gap `logic/` : toutes les fonctions ont consumer tests backend (travel-time, building-cost, production-rate, training-time).

### Changements

- `packages/shared/src/logic/building-cost.ts` : fusion des 2 imports, suppression des 2 casts `as number`.
- `packages/shared/src/logic/travel-time.ts` : suppression du cast `as number`.
