## Phase 2 — Refactor exécuté

### Nouveau helper (60L + 200L tests)

`battleforthecrown-backend/src/infra/pg-boss/queue-worker.helper.ts` expose :

- `registerScheduledQueueWorker(boss, logger, opts, handler)` — 4 workers cron sans data : `OyezWorker`, `WorldLifecycleWorker`, `ProductionWorker`, `CrownProductionWorker`. Log standardisé `<name> worker initialized (<cron> <tz>)`.
- `registerJobQueueWorker<TData>(boss, logger, opts, handler)` — 5 workers job-data : `TrainingWorker`, `ConstructionWorker`, `ReturnWorker`, `CombatWorker`, `ConquestFinalizeWorker`. Unwrap `jobs[0]` central, `workOptions` optionnels (utilisé par `ConstructionWorker.pollingIntervalSeconds: 1, batchSize: 1`).

Spec unit pure-logic (7 tests) : ordre des appels (`createQueue → work → schedule`), forwarding handler/data, passthrough `workOptions`, gestion d'erreur (rethrow + log).
