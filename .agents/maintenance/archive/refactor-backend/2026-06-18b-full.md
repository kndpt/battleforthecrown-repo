État repris du run 2026-06-18a — findings inchangés depuis le merge de

### Thème choisi

**D1 — Replace `getStrategyBonus(context)` switch by a typed
context→keys table**.

Pourquoi :
- 6 cases dans le switch (`combat/training/production/construction/storage/population`)
  font tous la même chose : `{ ...BASE_VILLAGE_STRATEGY_BONUS,
  [keys spécifiques au contexte]: getStrategyBonusValue(strategy, key) }`.
- Très faible risque : la projection reste pure ; pas de migration ; pas de
  nouvelle clé exposée ; comportement strictement identique
  (les keys non listées dans un contexte retombent sur les défauts via
  `BASE_VILLAGE_STRATEGY_BONUS`, déjà spread précédemment).
- Vérifiable : `yarn static-check`, suites unit existantes
  (`strategy-bonus.spec.ts`), spec dédiée pour la projection pure, smokes.
- Petit, bien borné, gagne ~45 lignes et rend l'ajout d'un futur contexte
  trivial (1 entrée dans la table).

## Refactor (Phase 2)

### Plan

1. Ajouter `battleforthecrown-backend/src/modules/strategy/strategy-bonus-projection.ts` :
   - `type StrategyBonusContext = 'combat' | 'production' | 'construction' | 'training' | 'storage' | 'population'`
   - `STRATEGY_BONUS_KEYS_BY_CONTEXT: Record<StrategyBonusContext, readonly (keyof StrategyBonus)[]>` — table type-safe.
   - `projectStrategyBonusForContext(strategy, context): Required<StrategyBonus>` —
     pure, déterministe ; itère sur la table, copie les bonus typés via
     un helper `assignBonus<K>` qui garde la corrélation key/valeur.
2. Refactor `VillageStrategyService.getStrategyBonus` :
   - Type du paramètre `context` → `StrategyBonusContext`.
   - Switch (45 lignes) → un appel `projectStrategyBonusForContext`.
   - Drop l'import `BASE_VILLAGE_STRATEGY_BONUS` (plus utilisé dans le service).
   - `getStrategyBonusValue` reste importé (toujours utilisé au
     `getStorageLimitForStrategy` ~ ligne 367).
3. Ajouter `strategy-bonus-projection.spec.ts` :
   - 1 test par contexte vérifiant les keys projetées + les défauts pour
     les keys non projetées.
   - 1 test boucle vérifiant `BALANCED` ⇒ tous les contextes retournent
     `BASE_VILLAGE_STRATEGY_BONUS`.
