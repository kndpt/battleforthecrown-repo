## Changes

### shared/combat/schemas.ts
- Added `CombatReportResponseSchema` (+ `CombatReportsResponseSchema`) — `z.object().passthrough()` to tolerate extra Prisma fields leaked by the backend presenter.
- Added `ScoutReportResponseSchema` (+ `ScoutReportsResponseSchema`).
- Added `ReinforcementReportResponseSchema` (+ `ReinforcementReportsResponseSchema`).
- Added `CaravanReportResponseSchema` (+ `CaravanReportsResponseSchema`) — moved from local `queries.ts`.

### shared/combat/dtos.ts
- Updated `CombatReportResponse` interface: added `worldId`, `attackerUserId`, `defenderUserId`, `createdAt`, and `captureFinalized.conquerorName`/`visualTier` fields that the backend actually returns but were missing from the shared contract.

### pixi/api/queries.ts
- Deleted local `CombatReportDto` interface (replaced with `type CombatReportDto = CombatReportResponse`).
- Deleted local `CombatLootDto` interface.
- Deleted local `CaravanReportResponseSchema` and `CaravanReportResourcesSchema` (moved to shared).
- Made `parseList`/`parseDetail` **required** in `ReportHooksConfig` (removed `?`).
- Removed 3 `as TList[]`/`as TDetail` trust-boundary bypasses in `createReportHooks`.
- Wired Zod parsers into all 4 report hook configurations.

### pixi/features/combat/combatReportView.ts
- Added `?? {}` fallback for `report.totalUnitsAttacker` (now optional in shared type).
- Used typed empty loot object for `resources`/`remaining` fallback instead of bare `{}`.
