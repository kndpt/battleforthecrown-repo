## Phase 2 — Refactor applied

**Theme:** Extract mutation invalidation helpers + fix recall-reinforcement cache gaps

### Changes

**`battleforthecrown-pixi/src/api/queries.ts`**
- Added `invalidateCombatDispatchQueries()` helper (shared by attack + scout mutations)
- Added `invalidateTroopMovementQueries()` helper (shared by reinforce + recall-reinforce)
- Refactored `useInitiateAttackMutation` to use `invalidateCombatDispatchQueries`
- Refactored `useInitiateScoutMutation` to use `invalidateCombatDispatchQueries`
- Refactored `useInitiateReinforceMutation` to use `invalidateTroopMovementQueries`
- **Fixed** `useRecallReinforcementMutation` to use `invalidateTroopMovementQueries` — adds missing `population`, `villagePower`, and `kingdomPowerPrefix` invalidations for both villages

**`battleforthecrown-pixi/src/api/mutationInvalidations.test.tsx`**
- Added test: reinforce mutation invalidates garrison + power for both origin and target
- Added test: recall reinforcement invalidates population + villagePower + kingdomPower for both villages

### Bug fix detail

`useRecallReinforcementMutation` was missing:
- `population(villageId)`, `population(originVillageId)` — troops leaving garrison changes population at both villages
- `villagePower(villageId)`, `villagePower(originVillageId)` — army composition changes affect power
- `kingdomPowerPrefix(userId)` — overall kingdom power changes
