## Looks bad but fine

- **crowns in applyUnitTrained** (F8 from agent audit): faux positif. Backend émet un événement `crowns.changed` séparé qui update le Zustand store directement. Pas besoin d'invalider le TQ crowns cache dans unit.trained.
- **applyCaravanRecalled missing targetVillageId resources**: faux positif. Un rappel se produit avant l'arrivée — les ressources cibles n'ont pas changé.
- **battle.resolved missing defenderVillageId**: design intentionnel. Événement séparé `village.attacked` gère le côté défenseur.
- **WorldMapScene.ts 804L factory pattern**: pas un god component React — factory Pixi avec séparation interne (ground, entities, expeditions, fog, vision).

## Phase 2 — fixes appliqués

Thème : **WS invalidation completeness + TQ staleTime hygiene + inline style extraction**

### ws-bindings.ts
1. `applyCaravanSent` +1 line: `queryKeys.resources(payload.targetVillageId)` — aligne avec REST `useInitiateCaravanMutation`
2. `applyVillageAttacked` +1 line: `queryKeys.resources(payload.defenderVillageId)` — les ressources pillées doivent être rafraîchies côté défenseur
3. `applyCaravanArrived` +1 line: `queryKeys.population(payload.targetVillageId)` — les porteurs occupent des slots de population

### queries.ts
4. `resourcesQueryOptions`: staleTime 0→5_000, supprimé `refetchOnMount:'always'` — aligné avec buildings/queue/population (tous 5s). Le Zustand store + WS `resources.changed` gèrent le rafraîchissement temps-réel.
5. `useWorldsQuery`: ajouté staleTime 30_000 — la liste des mondes est stable.
6. `usePublicWorldsQuery`: ajouté staleTime 30_000.
7. `useMyMembershipsQuery`: ajouté staleTime 10_000 — un peu plus volatile (join/leave/login).

### GameHeader.tsx + index.css

Suppression du `<style>` inline. Ajout des classes `.game-topbar-*` au media query `prefers-reduced-motion`.
