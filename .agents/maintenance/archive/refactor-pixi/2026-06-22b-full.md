## Refactor appliqué

**Thème : WorldMapScene memory safety + entity styling extraction**

### Bug fix

`buildWater` utilisait `requestAnimationFrame(buildWater)` sans stocker le handle. Si `exit()` était appelé avant le frame suivant (navigation rapide), `buildWater` s'exécutait sur une scène morte. Fix : handle stocké dans `waterRaf`, annulé dans `exit()`.

### Extraction

Module `worldMapEntityStyle.ts` (113L) — fonctions pures sans dépendance Pixi :
- `COLOR` (palette complète)
- `spriteSizeFor(entity)` — taille sprite par kind/tier
- `ownedHaloRxFactor(entity)` — rayon halo par castle tier
- `styleFor(entity)` — couleur, ring, radius, zIndex
- `aliasFor(entity)` — alias texture Assets
- Constantes : `BASE_PLAYER_SIZE`, `BASE_BARBARIAN_SIZE`, `WORLD_MIN_ZOOM`, `WORLD_MAX_ZOOM`, `ACTIVE_HALO_SCALE`

WorldMapScene.ts : 1027 → 930 lignes (-97).
