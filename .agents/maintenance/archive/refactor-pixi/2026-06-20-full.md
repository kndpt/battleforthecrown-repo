## Fixes applied

### F3 — WS resources invalidation for unit training

`applyUnitTrained` and `applyUnitTrainingCompleted` now invalidate `resources(villageId)`.
Unit training consumes resources; the REST mutation `useTrainUnitsMutation` already invalidated this key (queries.ts:675) but the WS handlers did not.

Tests updated: both test cases now seed and assert `resources(villageId)` invalidation.

### F16 — GPU texture leak on WorldMapScene exit

Added `fogContainer.cacheAsTexture(false)` in `exit()` before viewport cleanup.
Pixi v8 API: `cacheAsTexture(false)` releases the cached render texture. Without this, the GPU texture leaked every time the scene was exited and re-entered.
