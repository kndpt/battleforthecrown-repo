# refactor-pixi — 2026-06-19 full report

## Theme: report query factory extraction

### Audit dimensions

1. **queries.ts (1466L)**: 4 report types × 4 hooks (list, detail, markRead, delete) = 16 hooks with near-identical code (~230 lines duplicated). Caravan used Zod validation; others used raw typing. `useDeleteCaravanReportMutation` used `removeQueries()` instead of `invalidateQueries()` — inconsistent with the other 3 delete mutations.

2. **ws-bindings.ts (832L)**: Test coverage gaps for `applyScoutReported` (missing scout report prefix invalidation assertion), `applyRankingsChanged`, `applyVillageRemoved`. `resolveOrigin()` fallback `{x:0, y:0}` risk for new accounts. Handler calls not wrapped in try-catch.

3. **God components**: VillageView (530L, 6 boolean state flags), ArmyScreen (637L, 11 state vars), GameHeader (409L, inline `<style>` keyframe injection). SpecializedBuildingDetailModal, VillageViewSections, AttackDetailModal, CaravanLaunchModal are clean.

4. **Zustand stores**: resources.ts, crowns.ts, expeditions.ts duplicate TanStack Query cache — dual write via AuthenticatedShell effect + WS bindings. Architectural smell but large refactor (20+ files).

5. **Type debt**: Clean in production code. Only test fixtures use `as unknown as`. 2× `unknown[]` placeholders in CombatLoot interface.

6. **Pixi scenes**: Mostly symmetric enter/exit. WorldMapScene `fogContainer.cacheAsTexture()` has no explicit `uncache()` in exit(). Viewport not explicitly `destroy()`ed. 8 pixi files without test counterparts (BuildingSprite 294L and WorldMapScene 759L highest priority).

### Refactor applied

- **createReportHooks<TList, TDetail> factory**: Generic factory taking API paths, query keys, and optional Zod parsers. Instantiated for combat, scout, reinforcement, caravan. Same public export names preserved — zero caller impact.
- **removeQueries → invalidateQueries** fix for caravan delete mutation.
- **Result**: 1466 → 1360 lines (-106 net). 16 hooks → 4 factory instantiations + 1 factory (68 lines).
